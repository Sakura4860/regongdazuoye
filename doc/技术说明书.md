## 摘要

本方案针对城市集中供热换热站二次侧 DN100 主管 + 4–6 条支路的典型管网，提出一套"稳态水力数字孪生 + 轻量 AI 故障分类器"的在线测试与失调诊断方案。整个方案遵循 ISO/GUM 工程测试规范，涵盖测点布置、仪表选型、数据处理、误差分析与答辩口径。自写 Hardy-Cross 求解器作为数字孪生内核，通过对 5 支路小网络的 7 个扰动工况与 `scipy.optimize.fsolve` 独立求解器交叉验证，最大相对差异 0.0003 %。在仿真生成的 5900 条稳态样本上训练 XGBoost 故障分类器，5-fold 交叉验证 F1 macro = 0.963 ± 0.006。HII 综合水力失调指数的不确定度用 JCGM 101:2008 Monte Carlo 方法传递（N=10 000），可自动识别"跨阈值、判定置信不足"工况并延迟判定。

## 1. 工程背景

集中供热是北方城市冬季主导能源形式。换热站连接一次网（高温热水）与二次网（用户侧），其水力工况是否均衡直接决定用户舒适度、输配电耗与供热公平性。**水力失调**指部分支路流量偏离设计值、导致近端用户过热、远端用户供热不足，已被能源局《北方地区清洁供暖规划 2022》列为节能降耗首要症结之一。

传统水力平衡调节依赖运维人员经验，周期长、精度差、难以量化。**在线测试与智能诊断**因而成为新一代供热 SCADA 的核心功能。本方案设计在线测试硬件 + 数据处理算法 + 故障诊断模型的闭环，目标是：

- 以 ±2 % 的测量精度在线估计 5 条支路的流量/压差/温差；
- 用 HII 综合指数将"水力失调严重程度"量化为单个可读数字；
- 当判定置信不足时主动挂起诊断，避免误报；
- 当失调被确认时，自动定位到具体支路与故障类型。

## 2. 测试目标

1. **稳态水力观测**：实时采集二次供回水主管 + 5 条支路的温度（Pt100 A 级，±0.15 ℃）、压力（Rosemount 2088，±0.1 % FS）、流量（DN100 电磁，±0.5 %；DN50 一体式超声，±2 %）、阀位（电动阀自带反馈，±1 %）共 27 路信号，采样频率 1 Hz。
2. **失调辨识**：输出 HII ∈ [0, 1] 综合指数与四级分类（正常 / 轻度 / 中度 / 重度）。
3. **故障定位**：在失调被确认后，识别 5 类水力异常并定位到具体支路——单支路阻力↑（堵塞/阀关）、单支路阻力↓（阀过开）、双支路同时失调、泵频偏移、正常。
4. **不确定度报告**：每次诊断给出 HII 的 95 % 置信区间，并对跨阈值工况触发"延迟判定"。
5. **可复现工程产出**：说明书、PPT、代码仓库、附录表格齐备；算法经 5-fold CV 与独立求解器交叉验证。

## 3. 关键参数与传感器选型

### 3.1 必测 27 路信号

完整测点表见 `doc/附录/appendix_1_sensors.md`。按位置与类型分为 4 组：

- **主管（6 路）**：二次供/回水温度 T₁/T₂、供/回水压力 P₁/P₂、供回干管压差 ΔP、总流量 Q₀。
- **5 条支路的流量 Q₁–Q₅（共 5 路）**：支路 1–4 选 Kamstrup Multical 403 一体式超声热表；支路 5 选 E+H Promag 55S 插入电磁作为**在线比对基准**。
- **5 条支路的供/回水温度（共 10 路）**：Pt100 A 级铠装。
- **5 条支路阀位 θ₁–θ₅（共 5 路）+ 循环泵频率（1 路）**：电动阀 4–20 mA 反馈、变频器 Modbus RTU。

### 3.2 选型理由

- **温度选 Pt100 A 级**而非热电偶：二次侧温差量（ΔT ≈ 15 ℃）对热量计算关键，热电偶冷端补偿漂移与 A 级 Pt100 ±0.15 ℃ 最大允许偏差相比劣势明显；且 Pt100 三线制在现场总线环境下更成熟。
- **主管流量选 DN100 插入电磁**：能量平衡的基准量不能妥协精度。E+H Promag 50W ±0.5 %、100:1 量程比，是行业标配。
- **支路流量选 DN50 一体式超声**：详见 §5 方案比选。核心理由是"题目禁止大规模改造管网结构"——不允许支路开孔焊接法兰。
- **差压选 Rosemount 3051CD**：±0.075 % FS，量程 0–200 kPa，HART 协议便于系统集成。

### 3.3 间接计算量

由直接测量推导（不占独立测点）：

- 支路热负荷 Φᵢ = ρ c Qᵢ (T_s,i − T_r,i) —— 能量平衡校核；
- 支路比摩阻 R = ΔP / (L · Q²) —— 堵塞识别；
- 阀门 Kv = Q / √ΔP —— 阀位异常识别；
- 总流量平衡 Σ Qᵢ vs Q₀ —— 正常偏差 ≤ 3 %。

## 4. 测点布置（含灵敏度分析）

### 4.1 系统框图与测点位置

沿用图 1.1 的系统布置：二次供水主管从换热器出口到联箱接出 5 条并联支路，各支路串联用户换热器，再汇入回水联箱经循环泵回到换热器入口。测点位置满足：温度/压力/流量三类量均有配对、相邻支路间无干扰。

### 4.2 基于灵敏度驱动的测点优化

题目"在线测试"不等于"测得越多越好"。我们做如下灵敏度分析：

1. 用自写 Hardy-Cross 求解器构建数字孪生（`code/twin.py`、`code/solver.py`）；
2. 对每条支路施加 ±20 % 阻力扰动、对泵频施加 ±10 % 扰动，记录所有候选测点（14 维）的响应；
3. 得到 14 × 11 的相对灵敏度矩阵 S；
4. 用贪心 SVD 选择（每次选择使 `det(S_selected · S_selectedᵀ)` 最大的测点）选出**最小测点子集**。

> 图注：行为候选测点，列为故障工况；颜色越深绝对灵敏度越高。

> **[插入图 4.2]** `fig/sensor_selection.png` —— 候选测点信息量排序条形图
> 图注：红色为贪心选中的最小测点集，其余为冗余。

### 4.3 结论与冗余策略

结果显示：**3 个支路流量 + 2 个支路压差**足以唯一区分所有单点水力失调故障。但最终设计仍保留全 5 条支路流量测量，原因是：① 流量/温度配对完成能量校核；② 双支路同时失调时需要所有流量观测；③ 支路 5 的插入电磁可与相邻超声做月度比对校核，形成"在线校核链"。

## 5. 方案比选

### 5.1 候选方案

围绕支路流量测量，准备 4 个方案（原理、产品、精度、量程比、直管段、造价见 `doc/附录/appendix_2_scheme_comparison.md` 表 A）：A 一体式超声、A2 外夹式超声、B 插入电磁、C 涡街。

|------|------|---|----|---|---|

### 5.3 最终方案

- **主管 DN100**：插入电磁 B（能量平衡基准，精度不可妥协）；
- **支路 1–4 DN50**：一体式超声 A（经济性 × 足够精度 × 免维护）；
- **支路 5 DN50**：插入电磁 B（与相邻支路 4 超声做月度比对，累积偏差 > 3 % 触发校准）。

全案造价约 ¥128,800（见附录 2 表 E）。

### 5.4 答辩核心防御

- "为何不全部用电磁" —— 题目禁止大规模改造；HII 阈值 0.10 远大于 ±2 % 超声误差（MC 验证，见 §7）；TCO 更低。
- "为何保留 1 路电磁" —— 校核链 + 防止超声同源性漂移导致的系统性偏差。

## 6. 数据处理与失调诊断

双通道架构：**规则通道（HII 指数）** + **AI 通道（XGBoost 分类器）**。任一通道失效另一通道仍可工作。

### 6.1 HII 综合水力失调指数

定义（`code/hii.py`）：

```
θ_Q,i = (Qᵢ − Q_i,设计) / Q_i,设计       # 支路流量偏差
θ_P   = std{ΔPᵢ} / mean{ΔPᵢ}              # 支路压差分散度
θ_T,i = (ΔTᵢ − ΔT̄) / ΔT̄                   # 温差一致性
HII   = 0.5 · maxᵢ|θ_Q,i| + 0.3 · θ_P + 0.2 · maxᵢ|θ_T,i|
```

权重选择依据：流量是最直接量，压差反映支路不均分布，温度量滞后最大。四级分级：< 0.10 正常 / 0.10–0.20 轻度 / 0.20–0.35 中度 / ≥ 0.35 重度。

### 6.2 AI 故障定位

**训练数据**：用数字孪生注入 5 类扰动，共 5900 条稳态样本（正常 1500、单支路阻力↑ 1000、单支路阻力↓ 1000、双支路失调 1500、泵频偏移 600；另有 300 条"正常+大扰动"作为鲁棒性样本并入正常类）。每样本 20 维特征：Q₀、dP_header、dP_main、H_pump、Q₁–Q₅、dP₁–dP₅、dT₁–dT₅、ΣQ/Q₀。

> **[插入图 6.1]** `fig/eda_class_dist.png` —— 训练集各类故障样本量分布
> 图注：正常类 1800 条（含合并），各单点故障 1000 条以上，类别基本均衡。

> **[插入图 6.2]** `fig/eda_correlation.png` —— 20 维特征相关矩阵热图
> 图注：流量类、压差类、温差类呈现明显的分块结构，说明特征信息有冗余但不过度耦合。

**模型**：RandomForest baseline（n_estimators=200, depth=12）与 XGBoost 主力（max_depth=6, lr=0.1, n=300）。80/20 分层划分 + 5-fold CV。

**性能**：XGBoost 5-fold F1 macro = 0.963 ± 0.006，测试集 F1 = 0.957，每类 F1 ≥ 0.91；RandomForest F1 = 0.931（baseline 也达标）。

> **[插入图 6.3]** `fig/confusion_matrix.png` —— XGBoost 测试集混淆矩阵（行归一化）
> 图注：对角线均 ≥ 90 %，非对角主要误分发生在"双支路 vs 单支路"边缘工况，属预期内。

> **[插入图 6.4]** `fig/shap_summary.png` —— SHAP 特征重要性（跨 5 类平均 |SHAP|）
> 图注：单支路流量 Qᵢ 与支路压差 dPᵢ 对决策贡献最大，符合工程直觉。

### 6.3 推理流程

实时数据流：SCADA 1 Hz 采样 → EWMA 滑动滤波（§7.4） → 稳态判定通过后 → 一次 HII 计算 → 若 HII ≥ 0.10 → 调用 XGBoost 推理（`code/inference.py`）得到故障类型 + top-2 置信度 → 写入历史库并触发前端报警。

## 7. 误差分析与动态响应

### 7.1 不确定度来源

完整来源表见 `doc/附录/appendix_3_uncertainty.md` 表 A。主要来源：Pt100 ±0.15 ℃ 矩形（u=0.087 ℃），超声 ±2 % 矩形 + 1 % 三角安装偏差，电磁 ±0.5 %，3051CD 差压 ±0.075 % FS。

### 7.2 为什么用 Monte Carlo

HII 含 `max` 与 `std/mean` 等非线性、不可微分算子，违反 GUM 一阶线性传递前提。JCGM 101:2008 专门针对此类情形建议 Monte Carlo。实现上用 N = 10 000 次独立抽样（`code/mc_uncertainty.py`），对每个样本代入 HII 公式得到经验分布。

### 7.3 MC 结果与判定规则

对 5 个典型工况做 MC 传递：

| 工况 | HII 真值 | 95 % CI | 判定 |
|------|----------|---------|------|
| 正常工况 | 0.066 | [0.056, 0.080] | 可靠正常 |
| 支路 4 堵塞 × 2.15（阈值边缘） | 0.100 | [0.094, 0.111] | **跨越阈值，延迟判定** |
| 支路 4 堵塞 × 2.5 | 0.121 | [0.115, 0.132] | 可靠轻度失调 |
| 支路 2 阀门 55 % | 0.150 | [0.144, 0.160] | 可靠轻度失调 |
| 支路 2 阀门 30 % | 0.294 | [0.292, 0.301] | 可靠中度失调 |

> **[插入图 7.1]** `fig/mc_hist.png` —— 5 工况的 HII Monte Carlo 后验分布
> 图注：虚线为 0.10 / 0.20 / 0.35 三级阈值，红线为 95 % CI 上下分位。第二行左侧（支路 4 堵塞 × 2.15）的 CI 恰好横跨 0.10 阈值——规则：延迟 5 min 再判。

判定规则：若有任一分级阈值落在 CI 内 → 跨越阈值、延迟判定；否则按 CI 中位数所在等级给出"可靠 xx"结论。

### 7.4 动态响应处理

1. **EWMA 滑动滤波** α = 0.2：对支路 2 测量噪声 σ 从 0.199 → 0.069 m³/h，压低约 2.9 ×。
2. **稳态判定**：对 Q₀ 做 5 min 滑窗，σ/μ < 3 % 才进入稳态；非稳态挂起诊断。
3. **温度反卷积**：Pt100 时间常数 τ ≈ 30 s，`T_real ≈ T_meas + τ·dT/dt`，**先 EWMA 再反卷积**否则会放大噪声；补偿后温度误差 σ 从 0.260 → 0.209 ℃。
4. **工况归一化**：支路流量按 Q₀ 归一化，消除"总流量变化带来的伪失调"。

> **[插入图 7.2]** `fig/dynamic_response.png` —— 合成"阀门慢关"30 min 时序，4 子图展示滤波/反卷积/稳态判定/HII 时序
> 图注：橙色阴影为阀门过渡期 10–15 min；红色阴影为非稳态挂起诊断期；HII 时序的 EWMA 后曲线明显比原始信号稳定、更接近真值。

### 7.5 交叉验证

用 `scipy.optimize.fsolve` 把整个网络作为 7 元非线性方程组独立求解，与 Hardy-Cross 主求解器（brentq + 不动点迭代）对比。7 个扰动工况（正常 / 阀 55 %/30 % / 堵塞 ×2.5 / ×5 / 泵频 80 %/110 %）下，流量与压差最大相对差异 **0.0003 %**，远低于 plan 要求的 1 %。证明数字孪生实现的正确性不依赖特定数值方法。

## 8. 结论与展望

### 8.1 主要结论

1. 基于 Hardy-Cross 自写求解器（约 300 行 Python）+ scipy-fsolve 独立交叉验证，数字孪生可信度已验证。
2. HII 综合指数 + XGBoost 双通道诊断体系，测试集 F1 = 0.957，5-fold CV F1 = 0.963 ± 0.006。
3. Monte Carlo 误差传递给出 HII 95 % 置信区间，能自动识别跨阈值工况并延迟判定，比单阈值方案更工程化。
4. 支路流量测量方案经加权比选，最终采用"4 支路一体式超声 + 1 路插入电磁比对基准"组合，全案造价 ¥128,800。

### 8.2 未来工作

- 数据驱动部分尚未在真实 SCADA 历史上验证，后续可用换热站几周正常数据做 domain adaptation。
- 在二次网复杂拓扑（非纯并联）下的扩展：Hardy-Cross 需扩到多回路，后续可直接调用 PyDHN。
- 采用 Kalman 滤波 + PINN 做漏热检测（参考 Shi 2025），可作为下一代模型。
