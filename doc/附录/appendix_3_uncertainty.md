## B. 为什么选 Monte Carlo 而非线性传递？

HII = w_Q · max_i |θ_Q,i| + w_P · (σ_ΔP / μ_ΔP) + w_T · max_i |θ_T,i|

含 `max`、`std/mean` 等**非线性且不可微分**的算子，违反 GUM 一阶线性传递的前提。JCGM 101:2008 专门针对这种情形建议使用 Monte Carlo。我方实现：

- 输入分布：按表 A 各行抽样（N = 10 000）
- 正向模型：`compute_hii()` 对每个样本求 HII
- 输出：经验分布 → 2.5 % 与 97.5 % 分位得到 95 % 置信区间

## D. 判定规则（含置信度修正）

```
if   HII_hi < 0.10                              → 可靠正常
elif HII_lo > 0.35                              → 可靠重度失调
elif 某阈值 (0.10 / 0.20 / 0.35) 落入 [HII_lo, HII_hi] → 跨越阈值，延迟 5 min 再判
else                                             → 可靠 <level>
```

这一规则保证**跨阈值工况不会被误判**，延迟 5 min 后若仍未收敛则触发人工介入。

## E. 合成不确定度的关键观察

1. CI 宽度随 HII 增大**单调收窄**（0.024 → 0.009）。原因：θ_Q 分子项的相对误差随偏差主导，仪表绝对误差权重下降。
2. 对 HII 的主要贡献：支路超声 ±2 %（约 40 %） + 温度 ±0.15 ℃（约 25 %） + 其他（35 %）。
3. 若将支路 1 全换电磁（±0.5 %），CI 可进一步压缩 ≈ 30 %，但成本增加 ¥70 k，TCO 不经济（见附录 2）。

## F. 误差链路闭环

| 环节 | 做法 |
|------|------|
| 出厂校准 | 每件设备提供第三方校准证书（CNAS / ILAC） |
| 出厂后复检 | 投用前 168 h 通电老化 + 零点/量程校验 |
| 运行中比对 | 支路 5 插入电磁 vs 支路 4 超声 月度比对；|差异| > 3 % 触发检修 |
| 跨阈值判定 | MC 置信区间判定规则（见 §D）自动挂起诊断 |
| 追溯归档 | 每次诊断记录时间戳 + 输入分布参数 + 95 % CI，进 SCADA 历史库 |
