# 逐步执行表（Step-by-Step Checklist）

> 配合 `solution-plan.md` 使用。每条任务都有明确的输入 / 输出 / 验收标准，可按勾选执行。
> 总工作量约 **10–14 天**，1–2 人承担代码模块，其余组员并行推进文档与图表。
> 建议用 GitHub Project Board 或 Notion 看板管理。

## P1 数字孪生：自写 Hardy-Cross（Day 1–2）

| # | 任务 | 输入 | 输出 | 验收 | 预计耗时 |
|---|------|------|------|------|----------|
| 1.1 | 在 `code/twin.py` 实现节点-管段数据结构 `Pipe(id, L, D, K, branch_id)` | 无 | Python 类 | pytest 1 个样例过 | 1 h |
| 1.2 | 实现 Darcy-Weisbach 阻力：`hf = K·Q²`；`K = 8fL/(π²gD⁵)` | 管道参数 | `compute_K()` 函数 | 单元测试 K ≈ 理论值 ±1% | 1 h |
| 1.3 | 实现 Hardy-Cross 迭代求解器：按回路修正 ΔQ | 网络拓扑 + 总入口流量 | `solve_hardy_cross()` | 5 条支路测试网络收敛 ΔQ<1e-4 m³/h，迭代 <50 次 | 3 h |
| 1.4 | 建立题目工况实例：DN100 主管 + 5 支路（DN50），长度/设计流量按题目范围假设 | 文献参数表 | `networks/station_5br.py` | 设计工况 HII≈0 | 2 h |
| 1.5 | 加入换热器简化模型（温升/温降）和线性热容积 | T 边界条件 | `thermal.py` | 能量平衡误差 <1% | 2 h |
| 1.6 | 画出网络示意图（networkx + matplotlib） | `station_5br` | `fig/network.png` | 图中节点/支路清晰编号 | 1 h |
| 1.7 | 跟 PyDHN 结果交叉验证（可选） | 同一网络 | 对比表格 | 流量/压降相对差异 <5% | 4 h |

### 9.6 答辩必背问答（预设）

| 预设问题 | 标准回答要点 |
|----------|---------------|
| 为何不每条支路都装电磁？ | ①题目禁止大规模改造 ②一体式超声 Class 2 精度对 HII 阈值足够 ③TCO 更低 ④保留 1 条电磁做比对校核 |
| 仿真数据训练的模型能用吗？ | ①物理一致性由守恒律保证 ②同时保留 HII 规则通道 ③未来可做 domain adaptation |
| 为何用 XGBoost 不用 GNN？ | ①样本/特征规模下 XGBoost 饱和 ②SHAP 可解释性强 ③答辩对手组员能看懂 |
| 动态工况失调怎么办？ | ①稳态判定（5 min σ/μ<3%）挂起诊断 ②EWMA 滤波 ③温度反卷积 |
| 传感器漂移长期怎么处理？ | ①月度与基准电磁比对 ②差异>3% 触发校准 ③MC 误差分析已覆盖 |
| 为何 HII 权重是 0.5/0.3/0.2？ | ①流量是最直接指标 ②压差分散反映支路不均 ③温差滞后最大所以权重最小 ④灵敏度分析验证 |
| 成本大约多少？ | ①主管电磁 ¥20k ②支路超声 5×¥6k=¥30k ③差压/压力/温度 ¥15k ④控制柜+施工 ¥30k ⑤合计约 ¥95k |
